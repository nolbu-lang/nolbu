# 업무 서버 적용 가이드 (요약)

## 한 줄 요약

**관리직원에게 `bcjis-webapp.war` + DB 스크립트(`apply-indexes.ps1`, `patch-menu-budget-copy.sql`)를 주고, Tomcat WAR 교체·DB 스크립트 실행 후 재기동**하면 기존 DB에 연결된 채 개선 코드가 적용됩니다.

---

## 1. 기존 시스템에 무엇을 주면 되나?

### 관리직원 전달 목록

```
① bcjis-webapp.war          ← 애플리케이션 전체 (필수)
② deploy/ 폴더 또는 scripts/ 아래:
   - apply-indexes.ps1
   - patch-menu-budget-copy.sql
   - (loaddb 직후만) seed-comm-seq.sql
③ deploy/README-관리자용.md  ← 실행 방법
```

`globals.properties`는 **기존 서버 설정 그대로** 사용합니다. (DB URL·비밀번호 변경 없음)

---

## 2. 메뉴 — 이름은 그대로, 실행은 개선 코드

| 항목 | 내용 |
|------|------|
| 화면에 보이는 메뉴 | **기존 이름 유지** (예: 전년도예산/조서 적용[신규]) |
| 메뉴 URL | `/budget/budgetCopy.do` (기존과 동일) |
| 실제 동작 | **경량 조회 API** + **AccessLogFilter** + **DB 인덱스** 적용 |
| 개발용 메뉴 | `budgetCopyNew.do` URL 메뉴는 `patch-menu-budget-copy.sql` 로 **숨김** |

`/budget/budgetCopyNew.do` 로 접속해도 **같은 기존 화면**(`budgetCopy.jsp`)이 열립니다.

---

## 3. 화면 구성 — 기존 레이아웃 유지

| 구분 | 유지 내용 |
|------|-----------|
| 상단 | 조서구분(`condReportCd`) |
| 좌측 | **자료** (전년도) — 통계목·재원구분·부서 등 **기존 조건** |
| 우측 | **대상** — 동일 조건 + **적용** 버튼 |
| 그리드 | **기존 트리·컬럼** (요구/조정/전년도예산액 등) |
| 속도 | 조회 시 **경량 SQL** 사용 후, 화면 형식에 맞게 데이터 변환 |

※ 세세목만 조회하므로 트리는 **1단계(평면)** 로 표시됩니다. 컬럼·버튼·조건 UI는 기존과 같습니다.

---

## 4. 적용 순서 (관리직원)

```
1. 운영 DB 백업
2. patch-menu-budget-copy.sql 실행
3. apply-indexes.ps1 실행
4. (loaddb 직후만) seed-comm-seq.sql — DBA 협의
5. Tomcat 중지
6. webapps/bcjis-webapp.war 교체
7. Tomcat 기동
8. 로그인 → 메뉴 → 조회·적용 테스트
```

자동화: `deploy\deploy-all.ps1` (README-관리자용.md 참고)

---

## 5. 롤백

- WAR를 배포 전 백업본으로 되돌리고 Tomcat 재기동
- DB 인덱스·메뉴 변경은 롤백 필수 아님 (필요 시 DBA 협의)
