# 업무 서버 적용 가이드 (요약)

## 한 줄 요약

**관리직원에게 `bcjis-배포-YYYYMMDD.zip`(또는 WAR + scripts + deploy)을 전달하고, DB 스크립트 실행 → `globals.properties` AI 설정 추가 → Tomcat WAR 교체·재기동**하면 기존 DB에 연결된 채 **예산편성 개선 + AI 예산도우미**가 적용됩니다.

패키지 생성 (개발 PC):

```powershell
.\scripts\package-deploy.ps1
```

---

## 1. 관리직원 전달 목록

```
① bcjis-webapp.war                    ← 애플리케이션 (필수)
② deploy/                             ← 배포 스크립트·README
③ scripts/                            ← apply-indexes.ps1, patch-menu-budget-copy.sql 등
④ docs/운영배포_종합개선보고서.md      ← 종합 설명 (먼저 읽기)
⑤ deploy/globals.properties.ai-snippet.example  ← AI 설정 추가용
```

`globals.properties` **본문(DB 접속)** 은 기존 서버 파일 유지. AI 항목만 **추가**.

---

## 2. 개선 과제 2가지

| 과제 | 사용자 체감 |
|------|-------------|
| **예산편성시스템 개선** | 전년도예산/조서 적용 메뉴 **조회 속도 향상**, AJAX 안정 |
| **AI 예산도우미** | 모든 화면 하단 **「AI 예산편성 도우미」** — 심사조서 검색·내부 AI 질의 |

---

## 3. 메뉴·화면

| 항목 | 내용 |
|------|------|
| 전년도예산/조서 적용 | 메뉴 **이름·URL 동일**, 내부는 **경량 조회 API** 사용 |
| AI 챗봇 | **신규 UI** (하단 고정), 별도 메뉴 등록 불필요 (WAR 포함) |

---

## 4. 적용 순서

```
1. 운영 DB 백업
2. patch-menu-budget-copy.sql
3. apply-indexes.ps1
4. (loaddb 직후만) seed-comm-seq.sql — DBA 협의
5. globals.properties 에 AI 설정 추가 (ai-snippet.example)
6. Tomcat 중지 → WAR 교체 → 기동
7. 로그인 → 메뉴 조회 + AI 챗봇 테스트
```

자동화: `deploy\deploy-all.ps1` (DB+WAR, AI 설정은 수동)

---

## 5. AI 운영 전제

- WAS → `http://99.1.82.207:8080/llm-studio/.../budget_search` **네트워크 접근**
- 행정AI 담당부서와 **LLM Studio 엔드포인트·IP** 최종 확인
- 심사조서 검색 질문은 **DB 직접 조립**(LLM 미호출) — `AiReportDbOnly=true`

---

## 6. 롤백

- WAR 백업본 복원 + Tomcat 재기동
- DB 인덱스·메뉴 변경은 롤백 필수 아님
