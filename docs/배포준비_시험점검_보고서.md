## 1. 개요

### 1.1 개선 목적

기존 **전년도예산조서적용** 화면은 6단계 트리 전개·재원·통계목 등 무거운 조회로 인해 목록 로딩이 느리고, 매핑·일괄적용 UX가 부족했습니다.  
이번 개선은 **경량 평면 조회 + 매핑 목록 + 일괄적용** 중심의 신규 화면(`budgetCopyNew`)을 추가하고, 전역 AJAX 장애 원인(AccessLogFilter·TB_COMM_SEQ) 및 DB 인덱스 부재 문제를 함께 해결합니다.

### 1.2 변경 범위 요약

| 구분 | 파일(주요) | 변경 내용 |
|------|------------|-----------|
| UI | `budgetCopyNew.jsp`, `budgetCopyNew.js` | 평면 그리드, 매핑·일괄적용, 조건 UI 간소화 |
| Backend | `BudgetCopyNewController/Service/DAO` | 경량 조회·일괄적용 API |
| SQL | `BudgetCopyNew.xml` | `selectCopyNewMapList` 경량 쿼리 |
| 공통 | `AccessLogFilter.java` | JSON 파싱·tracelog 분리, 장애 격리 |
| DB 스크립트 | `create-indexes.sql`, `apply-indexes.ps1`, `seed-comm-seq.sql` | 성능 인덱스·채번 시드 |
| 배포 연동 | `import-cubrid-data.ps1` | loaddb 후 인덱스·시드 자동 실행 |

## 2. 개선 사항 상세

### 2.1 기능 — 전년도예산조서적용[신규]

| 기능 | 설명 |
|------|------|
| 경량 목록 조회 | 세세목(COMPO_LEVEL='1')만 평면 조회. 컬럼: **제목(DGRCOMPO_NM) + 조정액(ADJ_AMT)** |
| 이중 패널 | 좌: 전년도 예산 / 우: 적용대상(올해) |
| 매핑 목록 | 전년도 1건 + 올해 1건 선택 → 하단 매핑 그리드에 추가 |
| 중복 방지 | 동일 올해 세세목(teBgtCompoId) 중복 매핑 차단 |
| 일괄적용 | `ajaxBudgetCopyNewCopyReportBatch.do` — 한 트랜잭션에서 기존 `copyReport`+`copyPreInfo` 반복 |
| UI 정리 | 통계목·재원구분 필터 제거, 매핑 영역 확대·스크롤 |

**API 엔드포인트**

| URL | 용도 |
|-----|------|
| `/budget/budgetCopyNew.do` | 화면 |
| `/budget/ajaxBudgetCopyNewMapList.do` | 경량 평면 목록 |
| `/budget/ajaxBudgetCopyNewCopyReportBatch.do` | 매핑 일괄적용 |
| `/budget/ajaxBudgetCopyNewList.do` | (기존) 전체 트리 조회 — 호환 유지 |
| `/budget/ajaxBudgetCopyNewCopyReport.do` | (기존) 단건 적용 |

### 2.2 장애 수정 — AccessLogFilter / TB_COMM_SEQ

**증상:** 모든 AJAX에서 "조회에 실패하였습니다", JSON 파싱 오류

**원인**

1. `TB_COMM_SEQ` 비어 있음 → tracelog ID 채번 실패
2. `AccessLogFilter`가 tracelog INSERT **전에** 요청 본문을 소비하고, 예외 시 `JSON_PARAM` 미설정

**조치**

- `scripts/seed-comm-seq.sql`: TRACELOG 등 6개 채번키 시드
- `AccessLogFilter`: `JSON_PARAM` 선설정, tracelog 실패는 내부 try/catch 격리
- 파싱 전체 실패 시 빈 `JSONObject` attribute 보장

### 2.3 성능 — DB 인덱스

- `scripts/create-indexes.sql`: 약 **60개** CREATE INDEX (DGRCOMPO, REPORT, 차원 테이블, 권한/메뉴 등)
- `scripts/apply-indexes.ps1`: CUBRID `IF NOT EXISTS` 미지원 → **문장별 실행**, 중복은 건너뜀
- `import-cubrid-data.ps1`: loaddb 완료 후 인덱스·시드 자동 호출

**로컬 시험 (2026-06-02)**

| 항목 | 결과 |
|------|------|
| DB 인덱스 수 | 59개 |
| 경량 map 쿼리 (2026/010/1차) | **6,585건 / 0.28초** (csql) |
| apply-indexes.ps1 | 성공 0, 건너뜀 53, 실패 0 (이미 적용됨) |

### 2.4 소스 최적화 (2차)

- `budgetCopyNew.js`: ~1,350줄 → **~911줄** (미사용 formatter·콤보·중복 조회 로직 제거)
- `BudgetCopyNew.xml`: TB_DGRTEMNGMOK 중복 서브쿼리 → OUTER JOIN 1회, 불필요 SELECT 컬럼 제거
- Controller/Service/DAO: 공통 파라미터 처리, `copyReportBatch` → `copyReport()` 재사용

### 4.2 DB 작업 (loaddb/스키마 변경 없이 WAR만 교체하는 경우)

운영 DB가 **이미 가동 중**이면 아래만 실행:

```powershell
# 1) 인덱스 (중복은 자동 skip)
.\scripts\apply-indexes.ps1

# 2) 채번 시드 (최초 1회 또는 loaddb 후 필수)
csql -u bcjisapp -p <비밀번호> bcjis -i scripts\seed-comm-seq.sql
```

**주의:** `seed-comm-seq.sql`은 해당 table_name 행을 DELETE 후 INSERT합니다. 운영에서 이미 정상 채번 중이면 **DBA와 협의 후** 실행하세요.

### 4.3 애플리케이션 배포

```powershell
# 빌드
.\scripts\build.ps1

# WAR 백업
copy <tomcat>\webapps\bcjis-webapp.war bcjis-webapp.war.bak.<날짜>

# WAR 교체
copy target\bcjis-webapp.war <tomcat>\webapps\

# Tomcat 재기동
```
