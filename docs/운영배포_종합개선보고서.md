## 1. 개요

본 문서는 로컬 PC에서 개발·시험한 **예산편성심사정보시스템 개선본**을 실제 운영 서버에 배포하기 위해, **두 가지 개선 과제**의 내용·전달물·적용 절차를 **서버 관리자·업무 담당자** 관점에서 종합 정리한 보고서입니다.

### 1.1 개선 과제

| No | 과제 | 요약 |
|----|------|------|
| **1** | **예산편성시스템 개선** | 전년도예산조서적용 경량 조회·일괄적용, AJAX 전역 장애 해결, DB 인덱스 |
| **2** | **AI 예산도우미 추가** | 심사조서(경상·투자) 자연어 검색 + 부산시 내부 LLM Studio 연동 챗봇 |

### 1.2 배포 원칙

- **기존 운영 DB·접속 정보 유지** — loaddb·스키마 변경 없음 (WAR 교체 + DB 후처리 스크립트)
- **globals.properties** — Git 미포함, 서버별 기존 파일 유지 + **AI 설정만 추가**
- **전달 단위** — 소스 전체가 아닌 **WAR + 스크립트 + 문서** 패키지 (`scripts/package-deploy.ps1`)

## 2. 과제 1 — 예산편성시스템 개선

> 상세: `docs/개선사항_보고서.md`, `docs/배포준비_시험점검_보고서.md`

### 2.1 배경·문제점

| 영역 | 기존 문제 |
|------|-----------|
| 전년도예산조서적용 | 6단계 트리 조회로 **응답 지연**, 매핑·일괄적용 UX 부재 |
| 공통 AJAX | AccessLogFilter + TB_COMM_SEQ 미시드 → **전 메뉴 "조회 실패"** |
| DB | loaddb 후 **조회 인덱스 부족** |

### 2.2 주요 개선

| No | 개선 내용 | 효과 |
|----|-----------|------|
| 1 | 세세목 **경량 평면 조회** API (`ajaxBudgetCopyNewMapList.do`) | 목록 **0.28초/6,585건** 수준 (로컬) |
| 2 | 매핑 목록 + **일괄적용** API | 반복 적용 작업 감소 |
| 3 | 기존 `budgetCopy` 화면에 경량 API **연동** | 메뉴·UI **이름 유지**, 속도만 개선 |
| 4 | AccessLogFilter **tracelog 격리** + JSON_PARAM 선설정 | AJAX **전역 장애 해소** |
| 5 | `create-indexes.sql` + `apply-indexes.ps1` | 인덱스 **약 60개**, 중복 skip |
| 6 | `seed-comm-seq.sql` | loaddb 후 채번키 시드 |

### 2.3 변경 파일 (핵심)

## 3. 과제 2 — AI 예산도우미

> 상세: `docs/AI예산도우미_RAG구현보고서_및_하이퍼클로바연동방안.md`, `docs/AI예산편성도우미_사용설명서.md`

### 3.1 기능 개요

| 기능 | 설명 |
|------|------|
| **심사조서 RAG 검색** | 2013~ 경상·투자 심사조서 DB 검색, 정해진 보고 서식으로 답변 |
| **일반 질의** | 부산시 **내부 LLM Studio** (`budget_search` API) 연동 |
| **UI** | 로그인 후 **모든 화면 하단** 「AI 예산편성 도우미」 패널 |
| **보안** | LLM에 DB 직접 접근 없음, **SELECT만**, 세션 로그인 필수 |

### 3.2 아키텍처 (요약)

```
[챗봇 UI] → AiChatController → AiChatServiceImpl
                                  ├─ 심사조서 질문 → CUBRID 검색 (db-only, 빠름)
                                  └─ 일반 질문 → HyperClovaClient → LLM Studio API
```

### 3.3 내부 LLM API (인터페이스 정의서 연동)

| 항목 | 값 |
|------|-----|
| Method | POST |
| URL | `http://99.1.82.207:8080/llm-studio/v1/api/task/generate/syncapi/busan_ai_llm/budget_search` |
| Request | `user_query`, `datetime`, `stream`(Boolean), `dialog_history`(`"[]"`) |
| Response | `llm_result.answer` (stream=true 시 SSE, 서버에서 합쳐 표시) |

### 3.4 운영 설정 (globals.properties 추가)

`deploy/globals.properties.ai-snippet.example` 참고:

```properties
Globals.AiProvider = hyperclova
Globals.ClovaEndpoint = http://99.1.82.207:8080/llm-studio/.../budget_search
Globals.ClovaTimeoutMs = 120000
Globals.AiReportDbOnly = true
Globals.AiMaxReportBlocks = 50
```

### 3.5 검색 기능 (심사조서)

- 회계연도·경상/투자·사업명 키워드 검색
- `[소관부서]복지국` — 부서 필터
- `[검토내용]`, `[구분]` — 비정형 필드 검색
- 영문 **대소문자 무시** LIKE 검색
- 조정재원(국비·시비 등) **백만원** 표시

## 5. 운영 서버 적용 절차

### 5.1 사전 준비

- [ ] 배포 일정·담당자·롤백 담당 확정
- [ ] **운영 DB 백업**
- [ ] WAS → LLM 서버 **방화벽** (행정AI 담당 확인)
- [ ] 기존 `globals.properties` 백업

### 5.2 DB (1회·최초)

```powershell
.\deploy\deploy-db.ps1 -DbPassword "<비밀번호>" -RunMenuPatch
# loaddb 직후만: -RunSeed 추가 (DBA 협의)
```

### 5.3 애플리케이션

1. Tomcat **중지**
2. `webapps/bcjis-webapp.war` **백업** 후 교체
3. `webapps/bcjis-webapp/` 폴더 삭제 (잠금 시 프로세스 종료 후)
4. `globals.properties`에 **AI 설정 추가** (`deploy/globals.properties.ai-snippet.example`)
5. Tomcat **기동**

### 5.4 배포 후 확인

| # | 확인 항목 | 기대 |
|---|-----------|------|
| 1 | 로그인 | 정상 |
| 2 | 임의 메뉴 AJAX | SUCC, tracelog 오류 없음 |
| 3 | 전년도예산/조서 적용 조회 | 빠른 목록 표시 |
| 4 | AI 패널 | 하단 표시, **내부 AI** |
| 5 | AI: `부산시청 AI 예산 심의방법을 알려줘` | LLM **전문** 답변 |
| 6 | AI: `2025년 유가보조금 사업 찾아줘` | 심사조서 **표·요약** |
| 7 | catalina.log | ERROR 없음 |

### 5.5 롤백

- WAR 백업본 복원 + Tomcat 재기동
- DB 인덱스·메뉴 패치는 롤백 불필요
